# TUI 闪烁问题修复记录

## 背景

Bruce Coding Agent 在集成 Lanterna fullscreen TUI 时曾遇到过屏幕闪烁问题。历史修复的关键原则是：不要按固定帧率反复刷新屏幕，只在输入变化、消息追加、状态变化、审批弹窗变化或窗口 resize 时重绘。

本次给 RAG 索引增加输入框上方的进度状态栏后，闪烁再次出现，主要表现为：

- `/index` 索引过程中，状态栏持续更新时屏幕闪烁。
- 用户输入、删除字符时，屏幕也会闪烁。

## 根因

当前 `LanternaBruceRenderer.render()` 每次重绘都会执行 `screen.clear()`，然后重画消息区、补全浮层、RAG 进度状态栏、输入框和底部状态栏。

这在低频重绘时不明显，但两类场景会放大问题：

- 用户输入每个字符都会触发一次 render。
- RAG 索引进度每处理文件都会触发 `markDirty()`，索引文件较多时会形成高频 render。

`screen.clear()` 会把离屏 buffer 整屏清空，再重画全部内容。虽然最后只有一次 `screen.refresh()`，但在真实终端上整屏清空式刷新更容易产生可见闪烁，尤其是大屏、高频输入和索引进度密集更新时。

## 修复方案

本次修复采用两层策略：

1. 稳定终端尺寸下不再调用 `screen.clear()`

   Renderer 记录上一次的终端列数和行数。只有首次渲染或终端 resize 后才执行 `screen.clear()`。普通输入、删除、状态变化和 index 进度更新时，只在 Lanterna 离屏 buffer 中用空格覆盖画布，再绘制当前 UI，最后调用 `screen.refresh()`。

   这样保留了“本轮 render 能清掉旧字符”的语义，同时避免每次稳定帧都走整屏 clear。

2. RAG 索引进度做轻量节流

   `updateIndexProgress(...)` 会始终保存最新进度，但 `indexing` 阶段最多约每 100ms 触发一次 dirty render。`saving`、`done` 和清空状态栏不节流，确保阶段切换和完成后的 UI 清理及时显示。

## 结果

修复后：

- 输入新增、删除字符时仍即时更新，但不再每次整屏 clear。
- RAG 索引状态栏仍能持续展示最新进度，但不会对每个文件强制刷新一帧。
- 索引完成后，状态栏清空，输出面板只保留最终 `索引完成: ...` 摘要。

## 后续注意事项

- TUI 新增高频状态时，不要直接在每次事件里无条件触发整屏 render。
- 如果状态只用于展示最新值，应保存最新快照，并对 dirty render 做节流。
- resize、首次渲染、布局结构大变化可以使用 `screen.clear()`；普通输入和状态更新应避免稳定帧全屏 clear。
