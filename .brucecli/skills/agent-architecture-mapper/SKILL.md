# Agent Architecture Mapper

生成面向工程师的 Agent 架构说明，重点解释“请求如何流动、能力在哪里装配、状态在哪里保存、工具边界如何控制”。

## 工作流

1. 读取 `references/mapping-checklist.md`，按清单覆盖关键架构面。
2. 先定位入口和运行时装配：
   - CLI 入口、命令处理器、统一运行时。
   - Agent 模式分发点。
   - 工具注册表和能力开关。
3. 再沿一条用户请求画调用链：
   - slash 命令路径。
   - 普通任务路径。
   - ReAct、Plan、Multi-Agent 的分歧点。
4. 单独说明跨切面能力：
   - Memory 如何注入和保存。
   - RAG 如何索引、检索和暴露工具。
   - MCP/Web/Skill 如何注册工具。
   - HITL、并行和安全守卫在哪里生效。
5. 输出前检查图和文字是否能回答：
   - “新增一个能力应该接在哪里？”
   - “模型能看到什么上下文和工具？”
   - “哪些状态是会话级、任务级或持久化的？”

## 输出格式

优先使用简洁结构：

1. 一句话总览。
2. Mermaid 或 ASCII 架构图。
3. 请求调用链。
4. 模块职责表。
5. 关键状态与边界。
6. 新能力接入建议。

需要引用代码时给出文件路径和核心类名，避免铺满源码清单。除非用户要求，不要输出超过一张大型图。

## 风格

- 用中文解释，保持“学习型项目导览”的口吻。
- 先讲主干，再讲分支；先讲运行时，再讲具体能力。
- 明确区分事实、推断和建议。
- 不修改代码，除非用户明确要求实现架构变更。
