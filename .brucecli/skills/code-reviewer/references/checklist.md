# Code Review Checklist

## Correctness

- 检查空值、空集合、边界值、异常路径和状态转换。
- 检查调用方与被调用方的参数、返回值和生命周期约定是否一致。
- 检查新分支是否改变旧行为，失败时是否留下部分状态。

## Security and isolation

- 检查路径穿越、符号链接逃逸、命令注入和不可信内容执行。
- 检查权限边界、审批逻辑和只读/写入能力是否被绕过。
- 检查秘密、令牌和用户数据是否可能进入日志或错误信息。

## Concurrency and resources

- 检查共享可变状态、线程安全、竞态、死锁和顺序依赖。
- 检查线程池、流、数据库、进程和网络资源是否可靠关闭。
- 检查超时、取消和中断是否被正确传播。

## Compatibility and maintainability

- 检查公共接口、序列化格式、配置优先级和默认值是否兼容。
- 检查错误是否可诊断，降级路径是否会掩盖真实故障。
- 检查实现是否重复已有抽象或破坏包边界。

## Tests

- 检查正常路径、失败路径和关键边界是否有测试。
- 检查测试是否真正验证行为，而非仅验证方法被调用。
- 检查并发、安全和回归风险是否有针对性覆盖。
